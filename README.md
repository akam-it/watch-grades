# School Watching

Скрипт `watch_grades.py` собирает данные об оценках из School API, ищет долги по каждому ученику и формирует текстовый отчет по предметам.

## Что умеет

- Авторизуется в школьной системе для каждого ученика.
- Загружает карточку оценок.
- Определяет активную четверть.
- Считает долги и группирует их по предметам:
- Формирует итоговый отчет с суммой долгов.
- Отправляет отчет в Telegram (если включено).
- Не отправляет одинаковое уведомление повторно (сравнение с прошлым сообщением).

## Требования

- Python 3.11+
- Установленный пакет `requests`.

Установка зависимости:

```bash
python3 -m pip install requests
```

## Настройка

1. Скопируйте пример:

```bash
cp settings.toml.example settings.toml
```

2. Заполните `settings.toml`:
- `[[students]]`:
  - `name` - имя ученика в отчете
  - `login` - логин в школьной системе
  - `password` - пароль
- `subject_remove_tokens`:
  - массив строк, которые будут удаляться из названий предметов, если нужно (иногда, в названиях предметов может быть мусор, например у меня было яяяРусский язык)
- `[telegram]`:
  - `enabled` - включить отправку в Telegram (`true/false`)
  - `bot_token` - токен Telegram-бота
  - `chat_id` - ID чата/канала
  - `timeout` - таймаут запроса в секундах

## Как запустить

```bash
python3 watch_grades.py
```

После запуска скрипт:
- напечатает итоговый отчет в консоль;
- отправит сообщение в Telegram (если включено и настроено);
- обновит кэш-файлы в `cookies/`.

## Подключение Telegram-бота

1. Создайте бота через `@BotFather` и получите `bot_token`.
2. Добавьте бота в нужный чат/группу/канал.
3. Для канала: назначьте бота администратором с правом отправки сообщений.
4. Узнайте `chat_id`:
   - для публичного/приватного канала обычно начинается с `-100...`;
   - можно получить через `getUpdates` или любым удобным способом.
5. Заполните блок `[telegram]` в `settings.toml`.
6. Запустите скрипт и проверьте, что сообщение пришло.

## Структура файлов

- `watch_grades.py` - основной скрипт.
- `settings.toml` - рабочие настройки.
- `settings.toml.example` - пример настроек.
- `cookies/` - токены, кэш ответов и кэш последнего уведомления.

## Важно по безопасности

- Не публикуйте `settings.toml`: там логины/пароли и токен бота.
- Регулярно меняйте пароли и токены при утечке.
